Main Window

Upon successfully launching the application, you will see a window similar to this:



The numbered areas are as follows:

1) Menubar

Here you may access a number of menus; "File", "View", "Tools", and "Help". The "File" menu has a single menu item, "Quit", which you can click to exit the program. Next, the "View" menu has options "System Information" and "Running Processes"; the former spawns a dialog displaying basic information about your system, and the latter opens another dialog which shows a time line of recently physically executing tasks. The "Tools" menu has the entries "Simulate", to load the simulation module, and "Settings", which spawns a dialog allowing you to configure the program. Finally, the "Help" menu has items "Documentation" and "About"; the former option opens this user guide, while selecting "About" will open a dialog window displaying information about this software.

All of the menubar action can also be performed using hotkeys, such as Ctrl+Q to quit.

2) Pause/Resume Button & Status Information

Updating of the process information table, as previously described, can be stopped. This may be desirable to thoroughly view the table's contents. To do so, simply click the button with the pause symbol; the updating is now stopped indefinitely. Notice that the button's icon now changes from a pause symbol to a play symbol; if and when you wish to resume the updating, use the button again. Once resumed, the table and button will return to their initial states.

Adjacent to the pause/resume button you will notice status information; this describes the total number of active (including "zombie") processes on your system, and is updated at the same interval as the process table.

3) Process Information Table

This table is where you can monitor system tasks. Note that there are four headings to the table; "Process ID", "Name", "Status", and "Priority". For every process on your system, a row is added to the table with entries under each of these columns, with regard to the details of that process. By default, the table will update the information of its rows every three seconds, though this can be changed in the settings menu.

A brief description of the details displayed:

  1. Process ID: Numbers used to identify processes. Every process has a unique ID.
  2. Name: A name by which the task can also be identified, generally being the command which can be used to start it.
  3. Status: Last known state of the process; either Running, Sleeping, Disk Sleep, Stopped, or Zombie. These states all give an indication to the current activity of the process, and are defined by Linux; see the "proc" system manual for more information.
  4. Priority: The priority rating or "nice" value assigned to the process, between 20 and -20. A higher priority means the process is more likely to yield the processor to another process, while a lower priority means it is less likely.

A scrollbar will appear on the right-hand side of the window if it is not possible to display all processes in the available space. If this is so, you may drag the scrollbar to browse down further through the processes.

It is possible to sort the rows displayed by column; simply click on the respective column header to show in ascending order, or click twice for descending. For example, to list the processes in alphabetical order, click once upon the "Name" header. Sorting preferences can also be made permanent through the settings menu.

From the table, you may open the visualisation module, which allows you to manage the selected process and displays a visual representation of its state transitions. To open this module, navigate within the table to the process you want to view, and double-click anywhere within the table row.

4) Process Right-Click Menu

Right-clicking on any row in the process information table will open a context menu. This menu has several options, such as to view or terminate the process. Each action is relative to the process in the selected table row.
