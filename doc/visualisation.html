Process Viewer

Once launched, this module presents you with a dialog window. Information in this dialog is relative to the process that was selected from the main window.



This dialog contains several elements:

1) Process Information Table

This table shows precisely the same information as the table seen in the application's main window, however it is limited to displaying the information only for the particular process. Priority information is omitted, and instead shown below the table. Additionally, it updates by default more frequently than the other table; the state of the process is read and output to the table every second, though this can be changed in the settings menu.

2) Process State Model Diagram

The image shown here is a scalable diagram illustrating the traditional UNIX-like process state model. This model is significantly different from the one currently employed by Linux-based operating systems, though the concepts remain the same.

The image will show the process state highlighted in red which most closely relates to the actual state as defined by the system. This is updated every time the "State" cell is changed in the above table.

3) Set Priority Controls

To change the priority or "nice" value this task holds in the system, adjust the numeric value in the box shown here by pressing the up or down buttons, then click the "Set Priority" button to submit the request. This may be unsuccessful depending on your user permissions and system security policy; commonly, a normal user can increase priority for their own processes, but not lower. They generally cannot modify the priority of other users' tasks or system tasks whatsoever.

4) Stop/Resume Button

Pressing this button while it shows "Stop" will send the process a SIGSTOP signal, which will cause it to enter the Stopped state. If successful, you will then see the "Resume" button, which can be used to send the process a SIGCONT signal, urging it to resume execution. If this is also successful, the button will revert to "Stop". Again, success of these actions depends on your user permissions.

5) End Button

If permitted, this button sends the task a SIGTERM signal, requesting that it finish any immediate work and terminate.

6) Kill Button

Clicking this button will execute a KILL([pid], 9) system call, attempting to immediately end the appropriate process by sending it a SIGKILL signal. SIGKILL is more immediate and demanding than SIGTERM.

Note that this action should not be taken lightly. Ending any process in this way can indeed cause loss of data, among other, potentially more serious problems. It cannot be recommended to go through with this action unless the relevant process is completely unresponsive and the issue cannot be otherwise resolved. Even then, you are advised to consider the implications of continuing.

7) Close Button

Click this button to close the visualisation system, and return to the main window of the application.

8) Current CPU Display

If the task is currently executing on a processor, this area shows which particular CPU core is in use.

9) Record Button

Enables recording of the process, which continues until the user clicks the button again to stop recording. The recording can then be selected and played from "Manage Recordings" in the program's "View" menu.

10) Time Stamp

Displays the current system time, at each update.
