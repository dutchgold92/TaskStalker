Main Window

Upon successfully launching the application, you will see a window similar to this:

The numbered areas are as follows:

1) Menubar

Here you may access a number of menus; "File", "Tools", and "Help". The "File" menu has a single menu item, "Quit", which you can click to exit the program. The "Tools" menu has the entry "Simulate", to load the simulation module. Finally, the "Help" menu has items "Documentation" and "About"; the former opens this user guide, while selecting "About" will open a dialog window displaying information about this software.

2) Process Information Table

This table is where you can monitor system tasks. Note that there are three headings to the table; "Process ID", "Name", and "Status". For every process on your system, a row should be added to the table with entries under each of these columns, with regard to the details of that process. By default, the table will update the information of its rows every three seconds.

A scrollbar will appear on the right-hand side of the window if it is not possible to display all processes in the available space. If this is so, you may drag the scrollbar to browse down further through the processes.

It is possible to sort the rows displayed by column; simply click on the respective column header to show in ascending order, or click twice for descending. For example, to list the processes in alphabetical order, click once upon the "Name" header.

From the table, you may open the visualisation module, which allows you to manage the selected process and displays a visual representation of its state transitions. To open this module, navigate within the table to the process you want to view, and double-click on its "Process ID", "Name", or "State" table cell.

3) Pause/Resume Button

Updating of the process information table, as previously described, can be stopped. This may be desirable to thoroughly view the table's contents. To do so, simply click the button with the pause symbol; the updating is now stopped indefinitely. Notice that the button's icon now changes from a pause symbol to a play symbol; if and when you wish to resume the updating, use the button again. Once resumed, the table and button will return to their initial states.
